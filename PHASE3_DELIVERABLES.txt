================================================================================
  5. API ENDPOINTS ADDED
================================================================================

Endpoint 1: validate_payment
Validates winning bid amount and payment method
POST /api.php with action=validate_payment
Parameters: payment, item_number, winning_bid, item_value, reserve_amount
Returns: { bid_valid, payment_valid, valid, messages }

Endpoint 2: validate_category
Validates item category code
POST /api.php with action=validate_category
Parameters: category_code
Returns: { valid, category, message }

Endpoint 3: validate_bidder
Validates bidder registration data
POST /api.php with action=validate_bidder
Parameters: bidder { first_name, last_name, email, phone }
Returns: { valid, errors }

Endpoint 4: check_email_scan_cooldown
Checks if email scan is allowed (rate limit)
POST /api.php with action=check_email_scan_cooldown
Parameters: min_cooldown (optional, default 300)
Returns: { allowed, secondsUntilNext, message }

Endpoint 5: check_duplicate_email
Detects duplicate/similar email submissions
POST /api.php with action=check_duplicate_email
Parameters: description, donor_name, hours_window
Returns: { isDuplicate, similarEmail, message }

Endpoint 6: get_storage_report
Reports current storage usage
POST /api.php with action=get_storage_report
Parameters: none
Returns: { totalBytes, estimatedMB, quotaMB, percentageUsed, warning, message }

Updated: get_all
Now supports pagination
Parameters: page (default 1), limit (default 100, max 1000)
Returns: { data, page, limit, total, totalPages, hasMore }

Updated: get_all_data
Now supports pagination for items table
Parameters: page, limit, table (optional)
Returns: Paginated results with metadata
