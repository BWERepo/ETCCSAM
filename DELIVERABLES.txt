FEATURE HIGHLIGHTS
==================

Audit Logging
  - Every save operation logged (items, bidders, winners, payments)
  - Every delete operation logged with old_value
  - User ID, timestamp, IP address, success/failure tracked
  - Accessible via authenticated API endpoint with filtering
  - Queryable for compliance and forensics

PII Encryption
  - AES-256-CBC (NIST approved)
  - Bidder email and phone encrypted
  - Donor email and phone encrypted
  - Automatic encryption on write
  - Automatic decryption on read
  - Backward compatible (auto-encrypts unencrypted data)
  - Per-record random IV for security

Rate Limiting
  - Sliding window algorithm
  - Per-endpoint configuration
  - Default limits: 1/5min (email scan), 5/15min (login), 100/min (saves)
  - HTTP 429 Too Many Requests response
  - Retry-After header for client retry logic
  - Session-based (no distributed cache needed)

Server-Side Session Timeout
  - Replaces client-side only timeout
  - 30-minute idle timeout
  - Sliding window (resets on activity)
  - Returns 401 Unauthorized on timeout
  - Session fully destroyed on timeout

Debug Log Security
  - Direct file access blocked via .htaccess
  - Access only via authenticated API endpoint
  - Automatic rotation at 10MB
  - 30-day retention with auto-cleanup
  - Last N lines returnable via API

Soft Delete
  - 30-day grace period (GDPR compliant)
  - Records marked deleted with timestamp
  - SELECT queries automatically filter deleted records
  - Fully recoverable within 30 days
  - Auto-purge after retention period

Backup & Recovery
  - Automated daily backup creation
  - Stored in protected /backups directory
  - Gzip compressed SQL dumps
  - Named by timestamp: backup_YYYY-MM-DD_HH-mm-ss.sql.gz
  - Listable and verifiable via API
  - Manual restore capability

Payment Validation
  - Method whitelist: Cash, Check, Credit Card, Other
  - Amount validation: positive, 2 decimals, max $10k
  - Pre-insert validation
  - Failures logged to audit_log
  - Clear error messages for rejection reasons

DEPLOYMENT INSTRUCTIONS
=======================
1. Backup database:
   mysqldump -u user -p dbname > backup_pre_phase2.sql

2. Generate encryption key:
   php -r 'echo bin2hex(random_bytes(32));'

3. Update .env with:
   - ENCRYPTION_KEY (from step 2)
   - SESSION_TIMEOUT=1800
   - AUDIT_LOG_ENABLED=true
   - RATE_LIMITING_ENABLED=true

4. Deploy files:
   - security-helpers.php → /apps/sam/
   - api.php → /apps/sam/
   - .env → /apps/sam/
   - .htaccess → /apps/sam/
   - run-migrations.php → /apps/sam/

5. Create backups directory:
   mkdir -p /path/to/sam/backups
   chmod 700 /path/to/sam/backups

6. Run migrations:
   php run-migrations.php

7. Setup cron jobs (daily backup):
   0 2 * * * cd /path/to/sam && php run-backup.php

8. Test all endpoints:
   - Verify encryption (check database)
   - Verify audit logging (check audit_log table)
   - Verify rate limiting (send multiple requests)
   - Verify session timeout (wait 31 minutes)
   - Verify backup creation (check /backups directory)

See PHASE2_DEPLOYMENT_CHECKLIST.md for detailed step-by-step guide.
